## Phase 7: MaterialsolutionsNJ.com + David

### Purpose
Public-facing website for Material Solutions that integrates David (AI phone agent) for customer inquiries.

### Requirements

**Website Features:**
- Homepage with company branding
- Live inventory listings (auto-synced from Vortex app)
- Contact page with David's phone number
- About page (company story, team)
- Search/filter for inventory
- Individual item detail pages (specs, photos, price)

**David Integration:**
- Phone number prominently displayed
- Click-to-call functionality
- "Chat with David" widget (optional)
- Call tracking (log all conversations to database)
- Lead capture (name, company, phone, interest)

**Design:**
- Vortex Ventures neon theme (matching app)
- Mobile-responsive
- Fast loading (< 2 seconds)
- SEO-optimized (AEO strategy from MARKETPLACE_AUTOMATION_SPEC.md)

**Tech Stack:**
- **Option 1:** Next.js + Vercel (server-side rendering, great for SEO)
- **Option 2:** React + static hosting (simpler, cheaper)
- **Backend:** Share Railway backend with Vortex app (same database)

**Data Flow:**
```
Vortex App (inventory) → Railway DB → MaterialsolutionsNJ.com (public listings)
MaterialsolutionsNJ.com (leads) → David (phone calls) → Railway DB → Vortex App (lead management)
```

### Implementation Steps

1. **Design phase:**

   - Chris approves design
   - Delegate to Cipher for implementation

2. **Build core pages:**
   - Homepage
   - Inventory list

3. **David integration:**

4. **API integration:**
   - Create public API endpoint: `GET /api/public/inventory` (returns active, published items)
   - Website fetches inventory from Railway backend
   - Real-time sync (or cron job every 5 minutes)

5. **SEO/AEO optimization:**
   - Implement schema.org markup (Product type)
   - FAQ sections
   - Meta tags for each item
   - Sitemap generation

6. **Deploy:**
   - Domain: MaterialsolutionsNJ.com (Chris registers)
   - Hosting: Vercel (Next.js) or Netlify (static)
   - SSL certificate (automatic with Vercel/Netlify)

**Timeline:** 1-2 weeks

## Phase 8: Integration — Vortex App ↔ MaterialsolutionsNJ.com

### Purpose
Seamless data flow between internal tool (Vortex app) and public website.

### Key Integrations

**1. Inventory Publishing:**
- When Chris publishes inventory in Vortex app:
  - Item appears on MaterialsolutionsNJ.com automatically
  - SEO content (generated by Gemini) used for website listing
  - Photos uploaded in Vortex app → displayed on website

**2. Lead Capture:**
- Website visitor calls David → conversation logged
- Website contact form → creates lead in Vortex app
- David identifies inventory interest → links lead to specific item

**3. Status Sync:**
- Item sold in Vortex app → marked "SOLD" on website (or removed)
- Price changes in app → updated on website
- New photos added → synced to website

**4. Analytics Integration:**
- Track website traffic per inventory item
- Show Chris in Vortex app: "100 views, 5 calls" for each item
- Pipeline stage tracking: Listed → Website Views → Calls → Qualified → Sold

### Technical Implementation

**Backend changes:**
- Add `website_published` boolean to `inventory` table
- Add `website_views` counter (track clicks)
- Create webhook endpoint: `POST /api/webhooks/david-call`
  - Receives call data from Vocode
  - Creates lead in database
  - Links to inventory if mentioned

**Frontend changes (Vortex app):**
- Show "Website Status" in inventory detail modal
  - Published / Not Published
  - View count
  - Last updated timestamp
- Add "Unpublish from Website" button

**Frontend (MaterialsolutionsNJ.com):**
- Fetch inventory from `GET /api/public/inventory`
- Track page views via analytics (Google Analytics or Plausible)
- Send view events back to backend: `POST /api/inventory/:id/view`

**Timeline:** 1 week

## Phase 9: iOS App

### Purpose
Native iOS app for Vortex Forklift Sales Machine so Chris can manage inventory, leads, and pipeline from his phone.

### Requirements

**Core Features (Same as Web App):**
- Dashboard (stats overview)
- Inventory list + detail
- Add/edit inventory (photo upload from camera)
- Leads management
- Pipeline view
- David interface

**Mobile-Specific Enhancements:**
- Push notifications:
  - New lead captured
  - David completed a call
  - Item sold
- Camera integration:
  - Take photos directly in app
  - AI vision (identify forklift make/model)
- Call integration:
  - Tap lead's phone number → calls directly
  - Log call duration automatically
- Offline mode:
  - View cached inventory when no connection
  - Queue actions (e.g., add item) to sync later

### Tech Stack Options

**Option 1: React Native (Recommended)**
- Reuse existing React components (80%+ code reuse)
- Expo for rapid development
- Deploy to iOS (and Android if needed)
- Pro: Fast to build, code reuse
- Con: Not "native" feel (but close)

**Option 2: Swift + SwiftUI (Native)**
- Full native iOS experience
- Best performance
- Pro: Premium feel, Apple ecosystem integration
- Con: Rebuild from scratch (2-3 months)

**Option 3: Progressive Web App (PWA)**
- Wrap web app with Capacitor
- Install as "app" on iPhone
- Pro: Zero new code
- Con: Limited native features (no push notifications, camera less seamless)

**Recommendation:** Start with **React Native (Expo)** for speed, upgrade to native Swift later if needed.

### Implementation Steps

1. **Setup React Native project:**
   - Initialize Expo project
   - Copy components from web app
   - Adapt for mobile layout (smaller screens)

2. **Core screens:**
   - Dashboard
   - Inventory list
   - Inventory detail + edit
   - Add new inventory (with camera)
   - Leads list
   - Pipeline kanban (mobile-optimized)
   - David interface

3. **Mobile-specific features:**
   - Camera integration (react-native-camera or Expo Camera)
   - Push notifications (Expo Notifications + backend webhook)
   - Offline sync (AsyncStorage + queue)

4. **Testing:**
   - TestFlight beta (Chris tests on his iPhone)
   - Iterate based on feedback

5. **App Store submission:**
   - Apple Developer account ($99/year)
   - App icons + screenshots
   - Submit for review (~1 week approval)

**Timeline:** 3-4 weeks (React Native), 8-12 weeks (native Swift)
